DB SCHEMA SPECS

Multi-tenant, HIPAA-compliant database architecture engineered for zero-latency lookups.

PATIENTS SCHEMA

Primary master directory tracking patients, contact logs, and recall schedules.

COLUMN TYPE CONSTRAINT
id UUID PRIMARY KEY
clinic_id UUID FOREIGN KEY
first_name VARCHAR(100) NOT NULL
last_name VARCHAR(100) NOT NULL
phone VARCHAR(20) UNIQUE PER CLINIC
next_recall_due_date DATE INDEXED

APPOINTMENTS SCHEMA

Active schedule tracker restricted to 1-4 chairs for solo clinic capacity optimization.

COLUMN TYPE CONSTRAINT
id UUID PRIMARY KEY
clinic_id UUID FOREIGN KEY
chair_number INT CHECK (1 TO 4)
start_time TIMESTAMPTZ NOT NULL
end_time TIMESTAMPTZ NOT NULL
status VARCHAR(50) DEFAULT 'scheduled'

PRE_AUTHORIZATIONS SCHEMA

Kanban board records tracking insurance claims from submission to active treatment scheduling.

COLUMN TYPE CONSTRAINT
id UUID PRIMARY KEY
clinic_id UUID FOREIGN KEY
patient_id UUID FOREIGN KEY
treatment_description VARCHAR(255) NOT NULL
estimated_cost_cents BIGINT NOT NULL
status VARCHAR(50) INDEXED

SMS_REMINDERS SCHEMA

Queued recall text messages with delivery receipts synced via carrier webhooks.

COLUMN TYPE CONSTRAINT
id UUID PRIMARY KEY
clinic_id UUID FOREIGN KEY
type VARCHAR(50) NOT NULL
status VARCHAR(50) DEFAULT 'pending'
scheduled_for TIMESTAMPTZ INDEXED

    "chair_number": 3,

POSTGRESQL TRANSACTION
-- Step 1: Secure appointment availability
INSERT INTO appointments (clinic_id, chair_number, start_time, end_time)
SELECT '0a4f7000-8888-4444-bc84-99a38ff12026', 3, '2026-05-22 09:00:00-07', '2026-05-22 10:00:00-07'
WHERE NOT EXISTS (
    SELECT 1 FROM appointments 
    WHERE clinic_id = '0a4f7000-8888-4444-bc84-99a38ff12026' 
      AND chair_number = 3 
      AND start_time < '2026-05-22 10:00:00-07' 
      AND end_time > '2026-05-22 09:00:00-07'
) RETURNING id;
